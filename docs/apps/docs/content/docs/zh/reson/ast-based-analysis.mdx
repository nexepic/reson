抽象语法树（AST）分析的核心是“比较结构而非文本”，这使重复代码识别更加稳定。

## AST 示例

```c
int run() {
    for (int i = 0; i < 10; i++) {
        printf("Hello, World!\\n");
    }
    return 0;
}
```

对应的 AST 会保留语法结构信息（函数、循环、调用、返回等节点层级），从而可以在不同变量名、注释、字面量变化下继续匹配。

## 优势场景

### 1. 跨语言结构相似

C 与 C++ 代码在结构一致时，仍可识别重复逻辑。

### 2. 字面量变化容忍

字符串或常量变化不再轻易打断整段重复检测。

### 3. 注释噪声鲁棒

单行/多行注释差异不会显著影响结构匹配结果。

在大型仓库中，这种 AST 方式能明显减少“文本级误差”带来的噪声结果。
